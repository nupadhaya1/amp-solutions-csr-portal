# Review audit timeline

## Summary

The audit timeline helps CSRs understand prior account changes and support actions.

## CSR remediation steps

1. Open customer profile.
2. Review timeline events in chronological order.
3. Look for account edits, vehicle adds, payment failures, plan changes, cancellations, transfers, and support notes.
4. Use the timeline to explain what happened without over-sharing internal details.
5. Add a support note if the customer called about the history.

## Portal fields to check

- Audit event type
- Event timestamp
- Actor or CSR identity if available
- Related entity
- Support notes

## Escalate if

- Customer disputes an action.
- Event indicates fraud or unauthorized change.
- Timeline and current state conflict.

## Customer-safe explanation

I can review the account timeline to see when changes happened, such as payment failures, vehicle changes, plan changes, support notes, or cancellation events.
